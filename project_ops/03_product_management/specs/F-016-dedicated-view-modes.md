# F-016: Dedicated View Modes (Blogs Gallery, Courses Gallery, Events Calendar)

**Status:** âœ… Complete (Nov 30, 2025)

## Problem
Currently, all content types (Events, Courses, Blogs) use the same FeedList component with email grouping. This doesn't showcase the unique characteristics of each content type:
- **Blogs** would benefit from a visual gallery with rich previews
- **Courses** need a structured catalog view highlighting learning paths
- **Events** should display in a calendar format for temporal planning

## Goal
Create dedicated view components for each content type that:
1. Remove email wrapper grouping (show items directly)
2. Optimize layout for the content type's unique attributes
3. Maintain all existing features (tags, search, filters, source links)
4. Provide intuitive navigation and discovery

### Features
- **View Modes**: Day, Week, Month, Year (toggle buttons)
- **Navigation**: Previous/Next buttons, "Today" quick jump
- **Event Indicators**: Dots/blocks on calendar cells
- **Event List**: Below calendar showing selected day's events
- **Click to Expand**: Click date to see events for that day
- **Event Details**: Full EventCard in modal or side panel
- **Color Coding**: By event type or source
- **Drag Support** (future): Drag events to reschedule

### Component: `EventsCalendar.tsx`
